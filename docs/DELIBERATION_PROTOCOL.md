# 罗伯特规则 × Fishbowl × 组织行为学议事协议

本项目采用的是面向公共治理模拟的适配协议，而非声称复刻具有法律效力的正式议会程序。AI 角色的投票只表示模拟偏好分布，不能替代真实公众参与、法定授权或实地调研。

## 责任边界

| 层 | 职责 | 实现 |
| --- | --- | --- |
| 协议运行时 | 待决议案、附议、修正案优先级、结束讨论、模拟表决、事件账本 | `protocols/protocol_runtime.py` |
| Fishbowl 调度 | 内外圈、首发优先、支持/反对交替、低参与者优先 | `fishbowl.py`、`speaker_scheduler.py` |
| LLM 提示 | 角色语气、程序上下文、尊重互动、结构化程序动作 | `agents.py`、`sop_runtime.py` |
| 行为观察 | 发言支配、过早趋同、心理安全、信息共享与锚定风险 | `organizational_behavior/`、`observer.py` |
| 校验门 | 少数意见、证据缺口与程序就绪度 | `procedural_validators.py`、`validators.py` |

## 已执行的程序规则

1. 每次仅有一个待决事项；无关主议案不会覆盖当前议案。
2. 主议案和修正案默认都需要不同角色附议；附议不代表支持。
3. 修正案在主议案之前处理，默认最多两级。
4. 主持人和评审员不参与偏好表决。
5. 在少数意见未被保留或关键证据仍有缺口时，不得结束讨论并进入模拟表决。
6. 所有程序结果写入 append-only 的 `ProceduralEvent` 账本，并通过 SSE 推送至前端。

## 组织行为保护

- 发言支配：发言占比超过阈值时，建议主持人优先邀请低参与度角色。
- 群体迷思：检测到无异议的一致性表述时，要求提出失败条件或未满足底线。
- 心理安全：拦截明确的人身贬损，要求改写为针对观点、证据与后果的表达。
- 信息共享：低证据引用率会触发补充证据和数据来源的主持干预。
- 锚定/确认偏误：预算、费用、时间等参数缺乏替代比较时，要求反向证据或替代参数。

阈值位于 `configs/deliberation_protocol.yaml` 和 `configs/organizational_behavior.yaml`。修改配置前应同步补充协议测试，避免不经意改变程序公平性。
