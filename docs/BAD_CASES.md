# BAD_CASES.md — 已知失败模式与解决方案

## Bad Case 1: 角色漂移
**现象**: "环卫人员代表"突然开始从城市总体战略角度讲话
**原因**: LLM 在没有足够角色约束时倾向于使用通用的"专家"语气
**解决**: 
- [x] 每个角色 prompt 包含 can_say / cannot_say
- [ ] 正式版加入 Boundary Checker 实时判断发言是否越权
- [ ] 最终报告标记异常发言

## Bad Case 2: 观点坍缩
**现象**: 所有 Agent 很快都说"我同意大家，建议规范化管理"
**原因**: LLM 倾向于和谐，尤其在多轮对话后会自然趋同
**解决**:
- [x] 每轮要求角色保留不可让步底线
- [x] Observer 监测立场方差是否过快下降
- [ ] 正式版主持人可追问："你仍然不能接受什么？"

## Bad Case 3: 假共识
**现象**: 模型生成一个"所有人都接受"但实际不可执行的方案
**原因**: LLM 倾向于生成概括性、正面表述，缺少具体约束
**解决**:
- [x] 加入执行管理者和维护者角色
- [x] 要求方案包含责任主体、资金来源、执行成本
- [x] 报告中区分"AI 形成的文本共识"和"需要真实调研验证"

## Bad Case 4: 证据错配
**现象**: 摊贩引用了居民的投诉证据来支持自己
**原因**: 证据检索没有按角色过滤
**解决**:
- [x] Evidence Retriever 按 usable_agent 和 archetype 过滤
- [ ] 正式版用 Grounding Judge 检查"证据是否支持主张"

## Bad Case 5: 强势角色支配
**现象**: 街道办发言过多，环卫等沉默群体被边缘化
**解决**:
- [x] Orchestrator 优先调度发言较少者
- [x] Observer 实时展示发言占比
- [x] 设置每轮最大发言字数

## Bad Case 6: 模型刻板印象
**现象**: 摊贩被写成单一"可怜人"形象，忽视其能动性
**解决**:
- [x] 角色必须基于证据卡
- [x] 不允许用身份标签推断收入、教育程度、价值观
- [x] 报告中明确"AI 角色不等于真实群体"
