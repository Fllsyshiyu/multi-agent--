# DECISIONS.md 初稿

## D-001：先做模板生成而不是直接接 LLM

- 选项 A：直接调用 LLM，效果自然但成本高、不可复现、调试困难。
- 选项 B：先用模板生成，跑通系统链路，再替换为 LLM 发言器。
- 决策：选择 B。
- 理由：期中 Demo 的核心是证明系统架构可运行；模板生成能稳定产出 transcript、metrics、report，方便后续接入 Langfuse 和 eval。

## D-002：先做单议题夜市治理

- 选项 A：一次性覆盖 3 个议题。
- 选项 B：先深做一个议题，再扩展到电梯和共享单车。
- 决策：选择 B。
- 理由：夜市治理能同时展示生计、扰民、环卫、消费者、管理者和规划师，角色结构更完整。

## D-003：Observer 先实现 4 个轻量指标

- 发言公平性：按发言字数占比和 Gini。
- Grounding 率：含 evidence_id 的发言占比。
- 共识度：角色立场方差的反向指标。
- 回应网络：reply_to 边。

理由：这些指标能直接回应老师关于发言公平性、共识度、极化程度和有限轮数下表现的问题。
